¿Qué es una Ecodesarrolladora e Inmobiliaria?

Una ecodesarrolladora e inmobiliaria es una empresa que combina prácticas sostenibles con el desarrollo y la comercialización de propiedades. Se centra en crear y promover comunidades que respetan el medio ambiente, integrando principios de permacultura y bioconstrucción en su diseño y desarrollo, tales como ecoaldeas y parcelaciones permaculturales.

Este tipo de empresa busca minimizar el impacto ambiental mediante el uso de materiales naturales, la eficiencia energética y la gestión sostenible de recursos. Además, ofrece servicios de apoyo legal y tributario para facilitar transacciones seguras y cumplir con las regulaciones locales.

El objetivo principal es fomentar un estilo de vida sostenible y construir comunidades autosuficientes que estén en armonía con la naturaleza, permitiendo el acceso y la democratización de la tierra.

Componentes del Modelo de Negocio

Hipótesis de la propuesta de negocio

Asegurar una participación en el modelo de negocio de la ecodesarrolladora e inmobiliaria puede ser peligroso puesto que aún no se sabría cómo ejecutarlo, lo ideal es no generar falsas expectativas y ser muy claros.

Ahora, ¿Cómo se puede generar un modelo de negocio en esta construcción de MVP?

  1. Open Source: Lo primero es gestionar el código creado como Open Source abierto a los desarrolladores y personas de la comunidad en GitHub que permita contar con un producto validado para el portafolio de cada uno de los integrantes. No es un modelo de negocio directo, pero puede generar clientes potenciales para los servicios freelance.
  2. Servicios de desarrollo comunidad Ecodesarrolladora e inmobiliaria: Al contar con un producto culminado y validado, los aliados de Ecodesarroladores.com tendrán la posibilidad de acceder a los servicios del semillero en sus procesos de desarrollo.
  3. Publicidad: Tanto InteGrowHome como Vamos Pal Campo, que son las empresas creadoras de ecodesarrolladores.com, se encargarán de publicitar a los integrantes del semillero en sus diferentes canales digitales y tradicionales.
  4. Venta de producto: Si aparecen en el proceso clientes interesados en la tecnología, con una buena oferta en cuanto al pago, se puede vender el producto y dividir el ingreso en partes iguales según los integrantes que participen en el desarrollo del producto.

Requerimientos ✅

Módulos del Software

  • Registro y Login de Usuarios
  • Roles y Permisos (Administrador, Agente, Cliente)
  • Carga de Información de Lotes (Datos Generales, Ubicación, Precio, Tamaño, etc.)
  • Carga de PDFs y otros documentos (Imágenes, Planos, Documentos Legales)
  • Visualización y Edición de Información de Lotes
  • Filtro y Búsqueda de Lotes
  • Registro y Perfil de Clientes
  • Seguimiento de Intereses (Lotes Favoritos, Interacciones)
  • Historial de Contacto y Notas
  • Sección contenido de valor

Pasos para el Desarrollo

  • Reunir todos los requisitos específicos del negocio.
  • Mockups y Prototipos de la Interfaz de Usuario.
  • Diseño de la Base de Datos y Estructura de Datos.
  • Creación de componentes de interfaz.
  • Implementación de formularios de carga de datos y documentos.
  • Configuración de la base de datos.
  • Implementación de endpoints para CRUD de lotes, clientes y contenido de valor.
  • Integración de servicios de almacenamiento para PDFs y otros documentos.
  • Pruebas unitarias y de integración.
  • Asegurar la interoperabilidad entre frontend y backend.
  • Despliegue en un servidor o en la nube.
  • Monitoreo y mantenimiento continuo del sistema.

Funcionalidades Específicas

  • Formulario para ingresar datos básicos del lote.
  • Opción para cargar múltiples archivos PDF y otros documentos.
  • Subida de imágenes.
  • Sistema de validación para asegurar que todos los campos necesarios estén completos.
  • Creación de plantilla de información con la información subida de las propiedades.
  • Panel de administración donde los agentes pueden ver el historial de interacciones con cada cliente.
  • Sistema de notificaciones que alerta a los agentes cuando un cliente muestra interés en un lote específico.
  • Administración de contenido.

Arquitectura del Sistema

  • Lenguaje: Python
  • Frameworks: Django
  • Librerías: React
  • Base de Datos: MySQL
  • Integración con el Backend para la carga y gestión de archivos.
  • JWT (JSON Web Tokens) para Autenticación.
  • OAuth para Integración con Servicios Externos.

Participación en la Propiedad del Código

  • Ofrecer a los miembros del semillero una participación en la propiedad del desarrollo del código - Open Source en el cual todos puedan tener acceso desde Github.

Flujo de Procesos Desarrollo Seis Semanas

Cronograma de 6 Semanas para el Desarrollo del MVP

Semana 1: Definición y Prototipos

  • Lunes (Reunión Completa): Reunión inicial para reunir todos los requisitos específicos del negocio y definir el alcance del MVP.
  • Martes (Equipo Front): Sesión de trabajo en grupo para crear mockups y prototipos de la interfaz de usuario.
  • Jueves (Equipo Back): Revisión de requisitos y alineación sobre el diseño de la base de datos.
  • Tareas:
    • Documentar requisitos.
    • Crear mockups de al menos 3 pantallas clave (Inicio, Registro, Visualización de Lotes).

Semana 2: Diseño de Base de Datos y Configuración Inicial

  • Lunes (Reunión Completa): Revisión de los mockups y ajustes basados en el feedback del equipo.
  • Martes (Equipo Front): Diseño de la base de datos y estructura de datos (modelos relacionales en MySQL).
  • Jueves (Equipo Back): Configuración de la base de datos y discusión sobre la integración con el backend.
  • Tareas:
    • Definir tablas y relaciones necesarias.
    • Configurar el entorno de desarrollo con Django y React.

Semana 3: Desarrollo de Funcionalidades Básicas

  • Lunes (Reunión Completa): Creación de componentes de interfaz (registro, login, carga de información de lotes).
  • Martes (Equipo Front): Implementación de formularios de carga de datos y documentos.
  • Jueves (Equipo Back): Desarrollo de endpoints para el registro y autenticación de usuarios.
  • Tareas:
    • Desarrollar el formulario de registro y login.
    • Implementar la carga de datos de lotes y documentos.

Semana 4: CRUD y Integración de Documentos

  • Lunes (Reunión Completa): Implementación de endpoints para el CRUD de lotes y clientes.
  • Martes (Equipo Front): Integración de servicios de almacenamiento para PDFs y otros documentos.
  • Jueves (Equipo Back): Desarrollo y prueba de los endpoints CRUD.
  • Tareas:
    • Crear y probar los endpoints necesarios.
    • Implementar la lógica para la carga y visualización de archivos.

Semana 5: Interacción y Visualización

  • Lunes (Reunión Completa): Desarrollo de la funcionalidad de filtro y búsqueda de lotes.
  • Martes (Equipo Front): Implementación de la visualización de datos de clientes y lotes.
  • Jueves (Equipo Back): Ajustes finales en los endpoints para la visualización.
  • Tareas:
    • Crear la interfaz de visualización de lotes y clientes.
    • Implementar la lógica de búsqueda y filtrado.

Semana 6: Pruebas y Despliegue

  • Lunes (Reunión Completa): Planificación de pruebas unitarias y de integración.
  • Martes (Equipo Front): Realizar pruebas en los componentes de interfaz.
  • Jueves (Equipo Back): Realizar pruebas de los endpoints y ajustes finales.
  • Tareas:
    • Ejecutar pruebas en todo el sistema.
    • Preparar la documentación para el despliegue en el servidor.
